Core kubectl Commands

Get Information

kubectl get

List resources

# List all pods in the current namespace
kubectl get pods

# List pods with more details
kubectl get pods -o wide

# List pods in all namespaces
kubectl get pods --all-namespaces

# Get specific pod details in YAML format
kubectl get pod my-pod -o yaml

# List multiple resource types
kubectl get pods,services,deployments

kubectl describe

Show detailed information about resources

# Get detailed info about a specific pod
kubectl describe pod my-pod

# Get details about all services in a namespace
kubectl describe services

kubectl logs

Print container logs

# Get logs from a pod
kubectl logs my-pod

# Get logs from a specific container in a multi-container pod
kubectl logs my-pod -c my-container

# Stream logs in real-time
kubectl logs -f my-pod

Create and Modify Resources

kubectl create

Create resources from files or directly

# Create a deployment
kubectl create deployment nginx-deploy --image=nginx

# Create a namespace
kubectl create namespace test-namespace

kubectl apply

Apply configuration to resources

# Create or update resources defined in a file
kubectl apply -f deployment.yaml

# Apply configurations from multiple files
kubectl apply -f config1.yaml -f config2.yaml

# Apply configurations from a directory
kubectl apply -f ./configs/

kubectl delete

Remove resources

# Delete a pod
kubectl delete pod my-pod

# Delete all pods in a namespace
kubectl delete pods --all

# Delete resources using a configuration file
kubectl delete -f deployment.yaml

# Delete a namespace and all its resources
kubectl delete namespace test-namespace

kubectl edit

Edit resources directly

# Edit a deployment
kubectl edit deployment my-deployment

# Edit a service
kubectl edit service my-service

Pod Management

kubectl exec

Execute commands in containers

# Run a command in a pod
kubectl exec my-pod -- ls -l

# Get an interactive shell in a pod
kubectl exec -it my-pod -- /bin/bash

# Run a command in a specific container of a multi-container pod
kubectl exec -it my-pod -c my-container -- /bin/bash

kubectl port-forward

Forward local ports to pods

# Forward local port 8080 to port 80 in the pod
kubectl port-forward pod/my-pod 8080:80

# Forward to a service
kubectl port-forward svc/my-service 8080:80

Configuration and Context Management

kubectl config

Manage kubeconfig files

# Show current context
kubectl config current-context

# List all available contexts
kubectl config get-contexts

# Switch to a different context
kubectl config use-context my-context

# Set the default namespace for the current context
kubectl config set-context --current --namespace=test-namespace

Deployment Management

kubectl rollout

Manage rollouts

# Check rollout status
kubectl rollout status deployment/my-deployment

# View rollout history
kubectl rollout history deployment/my-deployment

# Rollback to previous version
kubectl rollout undo deployment/my-deployment

# Rollback to specific revision
kubectl rollout undo deployment/my-deployment --to-revision=2

# Restart a deployment
kubectl rollout restart deployment/my-deployment

kubectl scale

Scale deployments, replica sets, etc.

# Scale a deployment to 5 replicas
kubectl scale deployment/my-deployment --replicas=5

# Scale multiple deployments
kubectl scale deployment/deployment1 deployment/deployment2 --replicas=3

Advanced Operations

kubectl explain

Documentation for resources

# Get documentation for pods
kubectl explain pods

# Get documentation for specific field
kubectl explain pods.spec.containers

kubectl patch

Update specific fields

# Update a container image
kubectl patch deployment my-deployment -p '{"spec":{"template":{"spec":{"containers":[{"name":"my-container","image":"new-image:tag"}]}}}}'

kubectl top

Show resource usage

# Show CPU/memory usage for pods
kubectl top pods

# Show CPU/memory usage for nodes
kubectl top nodes

Common Use Cases

Deployment Workflows

  1. Create a new application deployment: 
    kubectl create deployment my-app --image=my-image:v1
kubectl expose deployment my-app --port=80 --type=LoadBalancer
  2. Update an application: 
    kubectl set image deployment/my-app my-app=my-image:v2
kubectl rollout status deployment/my-app
  3. Debug application issues: 
    kubectl logs deployment/my-app
kubectl describe pod my-app-pod-name
kubectl exec -it my-app-pod-name -- /bin/bash

Cluster Management

  1. Node maintenance (drain a node for maintenance): 
    kubectl drain my-node --ignore-daemonsets
# After maintenance:
kubectl uncordon my-node
  2. Namespace isolation (create resources in isolated namespaces): 
    kubectl create namespace project-a
kubectl -n project-a apply -f deployment.yaml
  3. Resource monitoring: 
    kubectl top nodes
kubectl top pods --all-namespaces

Troubleshooting

  1. Check pod health: 
    kubectl get pods
kubectl describe pod problematic-pod
kubectl logs problematic-pod
  2. Verify networking: 
    kubectl exec -it networking-test -- curl service-name
kubectl port-forward pod/my-pod 8080:80
# In another terminal: curl localhost:8080
  3. Check cluster events: 
    kubectl get events --sort-by='.lastTimestamp'

Creating a Kubernetes Simulator

1. Minikube - Local Kubernetes Cluster

Minikube is one of the simplest ways to run a full Kubernetes cluster locally for development and learning.

# Install Minikube
# On macOS:
brew install minikube

# On Linux:
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# Start a cluster
minikube start

# Check status
minikube status

Use case: Great for developers who want to test deployments on a real but isolated Kubernetes environment.

2. Kind (Kubernetes IN Docker)

Kind runs Kubernetes clusters using Docker containers as nodes, making it lightweight and easy to set up.

# Install Kind
# On macOS:
brew install kind

# On Linux:
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.17.0/kind-linux-amd64
chmod +x ./kind
sudo mv ./kind /usr/local/bin/

# Create a cluster
kind create cluster --name my-simulator

# Use a config file for multi-node clusters
kind create cluster --config=kind-config.yaml

Sample kind-config.yaml for a multi-node cluster:

kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
- role: worker
- role: worker

Use case: Great for CI/CD testing and multi-node simulations without heavy resource requirements.

3. K3d - Lightweight Kubernetes

K3d is a wrapper to run k3s (a lightweight Kubernetes distribution) in Docker.

# Install K3d
curl -s https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash

# Create a cluster
k3d cluster create my-simulator

# Create multi-node cluster
k3d cluster create my-simulator --agents 3

Use case: When you want something even lighter than Minikube but with most Kubernetes features.

Basic Docker Concepts

Docker is a platform that enables developers to build, package, and run applications in containers. Key components include:

  1. Images - Read-only templates used to create containers
  2. Containers - Running instances of Docker images
  3. Dockerfile - Text file with instructions to build an image
  4. Registry - Storage for Docker images (e.g., Docker Hub)
  5. Docker Compose - Tool for defining multi-container applications
  6. Docker Engine - Core software that runs and manages containers

Docker CLI Commands

Image Management

# List all images
docker images
docker image ls

# Pull an image from a registry
docker pull image_name:tag

# Build an image from a Dockerfile
docker build -t image_name:tag .

# Build with specific Dockerfile
docker build -t image_name:tag -f Dockerfile.prod .

# Tag an image
docker tag source_image:tag target_image:tag

# Remove an image
docker rmi image_name:tag
docker image rm image_name:tag

# Remove all unused images
docker image prune

# Remove all images not used by containers
docker image prune -a

Container Management

# Run a container
docker run image_name:tag

# Run with a custom name
docker run --name container_name image_name:tag

# Run in detached (background) mode
docker run -d image_name:tag

# Run interactively with terminal
docker run -it image_name:tag /bin/bash

# Run with port mapping
docker run -p host_port:container_port image_name:tag

# Run with volume mount
docker run -v host_path:container_path image_name:tag

# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# Start a stopped container
docker start container_name

# Stop a running container
docker stop container_name

Network Management

# List networks
docker network ls

# Create a network
docker network create network_name

# Create a network with specific subnet
docker network create --subnet=192.168.0.0/16 network_name

# Inspect a network
docker network inspect network_name

# Connect a container to a network
docker network connect network_name container_name

# Disconnect a container from a network
docker network disconnect network_name container_name

Volume Management

# List volumes
docker volume ls

# Create a volume
docker volume create volume_name

# Inspect a volume
docker volume inspect volume_name

# Remove a volume
docker volume rm volume_name

# Remove all unused volumes
docker volume prune

Docker Compose Commands

# Start services defined in docker-compose.yml
docker-compose up

# Start in detached mode
docker-compose up -d

# Stop services
docker-compose down

# Stop services and remove volumes
docker-compose down -v

# List running containers
docker-compose ps

# View container logs
docker-compose logs

# Execute a command in a service container
docker-compose exec service_name command

Dockerfile Reference

# Use an existing image as a base
FROM base_image:tag

# Set the working directory
WORKDIR /path/to/workdir

# Copy files from host to container
COPY source destination

# Run commands
RUN command

# Set environment variables
ENV key=value

# Document ports the container listens on
EXPOSE port

# Set default command to run
CMD ["executable", "param1", "param2"]

# Set entrypoint command
ENTRYPOINT ["executable", "param1", "param2"]

Ansible Basics

Ansible is an open-source automation tool that simplifies configuration management, application deployment, and task automation. It uses a declarative language to describe system configurations.

Installation

# Install Ansible on Ubuntu/Debian
sudo apt update
sudo apt install ansible

# Install Ansible on CentOS/RHEL
sudo yum install ansible

# Install Ansible on macOS
brew install ansible

# Check version
ansible --version

Basic Commands

# Run ad-hoc command on all hosts
ansible all -m ping

# Run ad-hoc command on specific hosts
ansible webservers -m shell -a "uptime"

# Run ad-hoc command with sudo
ansible all -m apt -a "name=nginx state=present" --become

# Check syntax of a playbook
ansible-playbook playbook.yml --syntax-check

# Run playbook
ansible-playbook playbook.yml

# Run playbook with variables
ansible-playbook playbook.yml -e "variable_name=value"

Playbooks

Playbooks are Ansible's configuration, deployment, and orchestration language. They express configurations, deployment steps, and orchestrate multi-machine deployments in a YAML format.

Playbook Structure

---
- name: My First Playbook
  hosts: webservers
  become: true
  vars:
    http_port: 80
    max_clients: 200
  
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present
    
    - name: Start nginx service
      service:
        name: nginx
        state: started
        enabled: yes
    
    - name: Create a directory
      file:
        path: /var/www/html/myapp
        state: directory
        mode: '0755'
    
    - name: Copy configuration
      template:
        src: templates/nginx.conf.j2
        dest: /etc/nginx/sites-available/default
      notify:
        - Restart nginx
  
  handlers:
    - name: Restart nginx
      service:
        name: nginx
        state: restarted

Common Modules

File Management

# Create a directory
- name: Create directory
  file:
    path: /opt/application
    state: directory
    mode: '0755'
    owner: app_user
    group: app_group

# Copy a file
- name: Copy configuration file
  copy:
    src: files/app.conf
    dest: /etc/app/app.conf
    owner: root
    group: root
    mode: '0644'

# Template a file
- name: Apply configuration template
  template:
    src: templates/app.conf.j2
    dest: /etc/app/app.conf
    
# Delete a file
- name: Remove temporary files
  file:
    path: /tmp/app-cache
    state: absent

Package Management

# Install packages (Ubuntu/Debian)
- name: Install required packages
  apt:
    name:
      - nginx
      - postgresql
      - python3-pip
    state: present
    update_cache: yes

# Install packages (RHEL/CentOS)
- name: Install required packages
  yum:
    name:
      - httpd
      - mariadb-server
      - python3
    state: present
    
# Install Python packages
- name: Install Python requirements
  pip:
    name:
      - django
      - requests
    state: present

Roles

Ansible roles are a way to organize playbooks and related files to facilitate sharing and reuse of content.

Role Structure

roles/
  └── webserver/
      ├── defaults/       # Default variables
      │   └── main.yml
      ├── files/          # Static files
      ├── handlers/       # Handler definitions
      │   └── main.yml
      ├── meta/           # Role metadata
      │   └── main.yml
      ├── tasks/          # Tasks
      │   └── main.yml
      ├── templates/      # Jinja2 templates
      └── vars/           # Role variables
          └── main.yml

Terraform Basics

Terraform is an infrastructure as code (IaC) tool that allows you to build, change, and version infrastructure safely and efficiently. It can manage popular service providers as well as custom in-house solutions.

Installation

# Install Terraform on Linux
curl -fsSL https://apt.releases.hashicorp.com/gpg | sudo apt-key add -
sudo apt-add-repository "deb [arch=amd64] https://apt.releases.hashicorp.com $(lsb_release -cs) main"
sudo apt update && sudo apt install terraform

# Install Terraform on macOS
brew tap hashicorp/tap
brew install hashicorp/tap/terraform

# Install Terraform on Windows with Chocolatey
choco install terraform

# Verify installation
terraform version

Basic Structure

Terraform configurations are written in the HashiCorp Configuration Language (HCL) in files with the .tf extension.

# main.tf - A simple example
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 4.0"
    }
  }
}

provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  
  tags = {
    Name = "example-instance"
  }
}

Workflow Commands

Terraform follows a workflow of initializing, planning, applying, and destroying infrastructure.

Initialization

# Initialize a new or existing Terraform configuration
terraform init

# Initialize and upgrade modules and plugins
terraform init -upgrade

# Initialize with a specific backend configuration
terraform init -backend-config=prod.backend.hcl

Planning

# Create an execution plan
terraform plan

# Save the plan to a file
terraform plan -out=tfplan

# Plan with variable values
terraform plan -var="instance_type=t2.small"

Applying

# Apply the changes required to reach the desired state
terraform apply

# Auto-approve apply without confirmation prompt
terraform apply -auto-approve

# Apply a saved plan
terraform apply tfplan

AWS CLI

The AWS Command Line Interface (CLI) is a unified tool to manage your AWS services from the command line.

Installation

# Install AWS CLI on Linux
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Install AWS CLI on macOS
brew install awscli

# Install AWS CLI on Windows (PowerShell)
msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2.msi

# Verify installation
aws --version

Configuration

# Configure AWS CLI with credentials
aws configure
# AWS Access Key ID: YOUR_ACCESS_KEY
# AWS Secret Access Key: YOUR_SECRET_KEY
# Default region name: us-west-2
# Default output format: json

# Configure a named profile
aws configure --profile project1

Compute Services

EC2 (Elastic Compute Cloud)

Amazon EC2 provides resizable compute capacity in the cloud. It is designed to make web-scale cloud computing easier for developers.

EC2 Instance Management

# Describe EC2 instances
aws ec2 describe-instances

# Describe specific instance
aws ec2 describe-instances --instance-ids i-1234567890abcdef0

# Create a new EC2 instance
aws ec2 run-instances \
    --image-id ami-0c55b159cbfafe1f0 \
    --instance-type t2.micro \
    --key-name MyKeyPair \
    --security-group-ids sg-903004f8 \
    --subnet-id subnet-6e7f829e

S3 (Simple Storage Service)

Amazon S3 is an object storage service offering industry-leading scalability, data availability, security, and performance.

Bucket Management

# List all buckets
aws s3 ls

# Create a bucket
aws s3 mb s3://my-bucket

# Delete an empty bucket
aws s3 rb s3://my-bucket

# Delete a bucket and all its contents
aws s3 rb s3://my-bucket --force
